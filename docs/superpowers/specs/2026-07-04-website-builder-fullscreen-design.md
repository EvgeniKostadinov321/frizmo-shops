# Уебсайт билдър — full-screen редактор + welcome модал

**Дата:** 2026-07-04
**Статус:** одобрен дизайн, готов за план
**Обхват:** `/dashboard/website` (редакторът на storefront-а)

## Мотивация

Редакторът на уебсайта („Уебсайт" таб в dashboard-а) наследява dashboard рамката
(`max-w-7xl` контейнер + 48px sidebar + padding), затова панелът и live preview-то
са притиснати в средата на екрана и не използват наличното пространство. На десктоп
това дава лош UX за най-визуалната задача в приложението.

Освен това няма насока към новия потребител, че билдърът е предназначен основно за
десктоп и че при желание екипът може да настрои сайта вместо него.

Тази промяна е **първата стъпка**. Пълен rebuild на билдъра (нови секции, inline
редакция върху preview-то, промени в схемата) е отделна бъдеща задача — извън обхвата тук.

## Цели

1. Full-screen редактор, който използва целия прозорец — голямо живо preview.
2. По-подреден ляв панел (табове вместо една дълга скролваща колона).
3. Welcome модал при всяко влизане (с „Не показвай повече"), който обяснява:
   - билдърът работи най-добре от компютър/лаптоп; работи и на телефон, но
     прегледът няма да е 1:1 с истинския сайт;
   - при желание екипът настройва целия сайт срещу имейл.
4. Замяна на емоджитата в билдъра с вътрешния `<Icon>` set (правило от
   CLAUDE-frontend.md: без емоджита в платформения UI).

## Извън обхвата

- Нови типове секции или промяна на `site_settings` схемата.
- Промяна на мутациите (`saveSiteSettings`, `savePreviewDraft`, `publishShop`,
  `unpublishShop`, `setShopLogo`) и на draft/live потока — остават непроменени.
- Inline редакция директно върху preview-то.
- Емоджитата в публичния marketing UI (cookie banner `🍪` и т.н.) — извън тази задача.

## Архитектура

### 1. Full-screen route

Редакторът се изнася в нова route група `(builder)` със собствен layout, за да
**не наследи** dashboard header/sidebar. Route групите не влияят на URL-а — пътят
остава `/dashboard/website`, затова nav линкът в dashboard-а не се променя.

```
src/app/(builder)/dashboard/website/
   ├── layout.tsx    ← full-screen layout (собствен топ-бар, без dashboard chrome)
   └── page.tsx      ← преместен от (dashboard)/dashboard/website/page.tsx
```

Старият `src/app/(dashboard)/dashboard/website/page.tsx` се изтрива.

`(builder)/layout.tsx` дублира auth guard-а на dashboard-а на root ниво (понеже
`(builder)` е отделна група и не минава през `(dashboard)/layout.tsx`): `getOwnShop()`
+ ако няма магазин → `redirect("/dashboard")`. Structure: `h-screen flex flex-col
overflow-hidden`.

### 2. Layout структура (десктоп)

```
┌──────────────────────────────────────────────────────────────┐
│ ← Табло   Уебсайт · {име}   [Чернова]     Отвори↗ Публик. Запази │  топ-бар h-14
├────────────────────┬─────────────────────────────────────────┤
│  [Тема][Секции][За нас]  │  [Начало][Продукти][За нас][Контакти] 🖥📱 │
│  ┌──────────────┐  │  ┌────────────────────────────────────┐ │
│  │ активен таб  │  │  │                                    │ │
│  │ (собствен    │  │  │        ЖИВО PREVIEW (iframe)        │ │
│  │  скрол)      │  │  │        (цял наличен екран)         │ │
│  └──────────────┘  │  └────────────────────────────────────┘ │
│   380px панел      │              1fr preview                  │
└────────────────────┴─────────────────────────────────────────┘
```

Топ-барът съдържа логиката за save/publish/dirty (изнесена от текущия `WebsiteEditor`).

### 3. Панел с табове

Левият панел (380px, fixed, собствен `overflow-y-auto`) има сегментиран контрол горе
с 3 таба (локален `useState`, **не** route — за да не се губи dirty състояние):

- **Тема** — ThemePanel (теми, палитри, собствени цветове, header layout) + „Текст
  във footer-а" + „Лого" (ImageUploader).
- **Секции** — „+ Добави" + SectionsList (drag & drop). Табът по подразбиране.
- **За нас** — „Представяне на бизнеса" (Textarea) + снимки (ImageUploader).

Съдържанието на всеки таб е същото като днешните Card-ове — само пренаредено в табове.

### 4. Мобилно поведение

Под `lg` панелът и preview-то не се събират един до друг. Табовете стават **4**:
„Тема / Секции / За нас / **Преглед**". Табът „Преглед" рендерира iframe-а на цял
екран (панелът се скрива); другите три показват панела. Device toggle (desktop/mobile)
се показва само в „Преглед" таба.

### 5. Welcome модал

Client компонент (`website-intro-modal.tsx`), монтиран в `(builder)/layout.tsx`.
Показва `<Modal>` ако `localStorage["frizmo-website-intro"]` липсва — при **всяко**
влизане, докато потребителят не чекне „Не показвай това повече" (същият паттерн като
cookie banner-а: `queueMicrotask` проверка за да не чупи SSR hydration).

Съдържание (BG, типографски кавички „…"):

- Заглавие: **Добре дошъл в редактора на твоя сайт**
- Блок 1 (икона `monitor`): **Редакторът работи най-добре от компютър или лаптоп** —
  за удобна и прецизна настройка препоръчваме голям екран. Работи и от телефон, но
  заради малкото пространство прегледът няма да изглежда 1:1 с истинския ти сайт.
- Блок 2 (икона `sparkles`): **Искаш готов сайт без усилия?** — свържи се с нас на
  **supportfrizmo@gmail.com** (`mailto:` линк) и ще настроим целия ти сайт вместо теб.
- Долу: Checkbox „Не показвай това повече" + Button „Разбрах, да започваме".

Затваря се само през бутона (Modal не се затваря при клик отвън, по конвенция). Ако
чекбоксът е бил маркиран при затваряне → записва ключа в localStorage.

### 6. Замяна на емоджита

Билдър UI-ят ползва емоджита в разрез с правилото. Заменят се с `<Icon>`:

| Място | Емоджи | Нова икона |
|---|---|---|
| SectionsList — дръжка | `⠿` | `grip-vertical` |
| SectionsList — видимо/скрито | `👁` / `🚫` | `eye` / `eye-off` (вече ги има) |
| SectionsList — редакция | `✎` | `pencil` (вече) |
| SectionsList — премахни | `✕` | `x` (вече) |
| SectionForm — премахни ред | `✕` | `x` (вече) |
| Preview — device toggle | `🖥` / `📱` | `monitor` / `smartphone` |
| Section picker + SECTION_DEFS.icon | 13 емоджита | 13 `<Icon>` имена (виж таблица) |
| Modal — close бутон | `✕` | `x` (вече) |

**Нови икони за `icon.tsx`:** `grip-vertical`, `monitor`, `smartphone`, `sparkles`,
и секционни: `layout-panel` (hero), `megaphone` (announcement), `star`
(featured-products), `grid` (category-grid), `tag` (promo-banner), `image-text`
(image-text — reuse `image`), `text` (rich-text), `quote` (testimonials),
`shield-check` (trust-badges), `images` (gallery), `help-circle` (faq), `map-pin`
(contact-map), `link` (socials).

`SECTION_DEFS.icon` сменя типа от `string` (емоджи) на `IconName`. `BADGE_LABELS` в
section-form.tsx (с емоджита в текста) става само текст (или с малка водеща `<Icon>`).
