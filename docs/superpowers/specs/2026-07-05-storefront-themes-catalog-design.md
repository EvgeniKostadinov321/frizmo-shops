# Storefront теми — разширен каталог (дизайн)

**Дата:** 2026-07-05
**Статус:** одобрена посока (спец) — визуалната посока валидирана в browser brainstorming
**Обхват:** разширяване на storefront темите от 3 на 9, вкл. тъмни теми
**Зависимост за:** onboarding setup wizard (спец 2026-07-05) — Стъпка 1 (избор на тема)
**База:** проучване `docs/research/2026-07-05-ecommerce-design-research.md`

## Мотивация

Днес има 3 storefront теми (`classic`/`modern`/`warm`, всички светли) в
`src/lib/themes.ts`. Проучването на 40+ реални магазина + трендове 2026 показа, че
всеки вертикал има ясно настроение, а някои (premium козметика, бижута) изискват
**тъмна** естетика. Разширяваме до **9 теми** (вкл. 3 тъмни), всяка с вертикално
предназначение — точно както Shopify курира теми по индустрия.

Визуалната посока е валидирана през browser mockup-и (пълен storefront на „Ателие"
топла светла + „Оникс" тъмна premium — одобрени от собственика).

## Ключово архитектурно решение (от предишен брейнсторминг)

**Тъмните теми са ОТДЕЛНИ теми, не light/dark превключвател.** Търговец избира
„Оникс" (тъмна) или „Ателие" (топла светла) — темата носи цялото си усещане,
включително дали е тъмна. Нула превключватели, нула двойно тестване. Storefront-ът
остава без глобален theme toggle; всяка тема просто задава своите `--sf-*` стойности
(тъмните теми = тъмен фон/светъл текст в самите токени).

Това НЕ променя storefront архитектурата — `themeStyle()` вече излива `--sf-*` като
CSS variables; тъмните теми просто носят тъмни стойности. Секционните компоненти вече
четат само `--sf-*`, значи работят с тъмни теми без промяна по тях.

## Палитри (референтни — финалните се калибрират при импл.)

Всяка тема дефинира пълния `ThemeVars` набор (`--sf-bg`, `--sf-surface`, `--sf-text`,
`--sf-muted`, `--sf-border`, `--sf-radius`, `--sf-heading-weight`, `--sf-font-heading`,
`--sf-font-body`) + `--sf-primary`/`--sf-accent` от избраната палитра. Референтни ядра
(от mockup-ите + проучването, hex 2026):

- **Ателие:** bg `#faf5ec`, surface `#fffdf8`, text `#33271a`, primary `#a9642e`
  (глина/теракота), border `#e9ddc9`, radius `0.625rem`. (≈ наследява днешната „warm")
- **Витрина:** bg `#ffffff`, surface `#fafafa`, text `#111`, primary тъмен/черен,
  radius малък (`2px`), uppercase display заглавия. (≈ днешната „modern")
- **Пулс:** bg тъмен `#111`, text бял, accent неон (`#e8ff45` или подобен), тежки
  заглавия. Тъмна.
- **Ефир:** bg `#fdf1ef`→`#f6e7ef`, text `#5a3d47`, primary приглушено розово
  `#c98a9b`, мек сериф.
- **Оникс:** bg `#14100c`, surface `#1c1610`, text `#f3ead9`, accent злато `#c9a25a`,
  border `#2e2519`, radius `4px`, висок-контраст display сериф. ТЪМНА.
- **Сигнал:** bg `#f4f6f8`, text `#0f1b2a`, primary teal `#0e7490`, radius `6px`,
  bouba grotesk. По подразбиране включва trust-badges + faq в набора секции.
- **Основа:** bg `#f5f2ee`, text `#211d18`, accent амбър `#d98a1e`, кондензиран
  шрифт, ляв акцентен борд. Индустриална светла.
- **Гранит:** bg `#1c1e21`, surface `#2a2d31`, text `#eef0f2`, accent амбър `#f0a63c`,
  кондензиран шрифт. ТЪМНА индустриална.
- **Класик:** сегашната неутрална (Inter, бяло/сиво). Без промяна.

## Шрифтове (решение: 3–4 споделени, не по двойка на тема)

Темите СПОДЕЛЯТ малък шрифтов набор (по-лесна поддръжка; всяка тема пак зарежда само
своите две — performance е еднакъв). Вече заредени в `layout.tsx`: Inter,
Space Grotesk, Lora + (платформени) Sofia Sans, Onest, Sofia Sans Condensed.

Нужни за новите теми (добавят се към `next/font`):
- **Display сериф** (за Оникс premium; висок контраст) — напр. Playfair Display.
- **Кондензиран сан** (за Основа/Гранит индустриални) — напр. Oswald / Archivo Narrow.
- (по избор) топъл сериф за Ателие вместо Lora, ако Lora не носи усещането.

Мапинг тема → (heading, body):
- Ателие → Lora (или топъл сериф) / Inter
- Витрина → Space Grotesk (или Inter tight) / Inter
- Пулс → тежък гротеск (Space Grotesk 700) / Inter
- Ефир → мек сериф (Lora) / Inter
- Оникс → Playfair Display / Inter
- Сигнал → Space Grotesk / Inter
- Основа → Oswald / Inter
- Гранит → Oswald / Inter
- Класик → Inter / Inter

⚠️ Всички шрифтове с `subsets: ["latin","cyrillic"]` където е възможно (BG текст).
Playfair и Lora имат cyrillic; Oswald има cyrillic; проверява се при импл. Ако
кондензиран cyrillic липсва → алтернатива (напр. PT Sans Narrow има cyrillic).

## Метаданни на всяка тема (за wizard preview картите)

Всяка тема получава метаданни (нов obект до `THEME_PRESETS`):
- `label` (име, напр. „Ателие"),
- `tagline` (усещане, напр. „топла, автентична — за занаяти и храни"),
- `bestFor` (кратък списък категории),
- `isDark` (boolean — за групиране/иконка в wizard-а),
- примерно hero съдържание по вертикала (за живия preview в Стъпка 1).

## Технически детайли (какво се пипа)

- `src/schemas/site-settings.ts`: `THEMES` enum разширяване от 3 → 9. **Толерантният
  parse** вече пази стари магазини (невалидна тема → default), но добавяме и explicit
  fallback: премахнат/преименуван theme id → `classic`.
- `src/lib/themes.ts`: `THEME_PRESETS` +6 записа (пълен `ThemeVars` + тъмни стойности);
  `THEME_LABELS` +6; нов `THEME_META` (tagline/bestFor/isDark); нов
  `CATEGORY_THEME_RECOMMENDATIONS` мапинг.
- `src/app/layout.tsx`: +2–3 нови `next/font` (display сериф, кондензиран).
- Storefront секциите: **без промяна** (четат само `--sf-*`). Задължителен визуален
  pass на 13-те секции × 9 теми (light + тъмните).
- ThemePanel (в редактора): вече показва теми — разширява се да показва 9 (може с
  групиране светли/тъмни); palette presets остават.

## Тестове

- Unit: `themeStyle()` връща валиден CSSProperties за всяка от 9-те теми; всеки
  theme id в `THEMES` има запис в `THEME_PRESETS`/`THEME_META`; всяка категория има
  ≥2 препоръки, всички сочещи валидни теми.
- Визуален pass (собственик): всяка тема × ключови секции, light + тъмни, 375px.
- `pnpm check` зелено.
