# N9 + N12 — Подаръчна опаковка + Връщания (спец)

> Одобрен дизайн 2026-07-08. Ред: **N9 → N12**, всеки с гейт (`pnpm check`) +
> commit. Потвърдено: (1) N9 е настройка на търговеца, изключена по
> подразбиране; (2) N12 срок за заявка = **избор на търговеца: 14 / 30 / 45
> дни** от „Завършена" (дефолт 14 — законовият минимум); (3) `returned` се
> изключва от приходите навсякъде, както `cancelled`.

---

## N9 — Подаръчна опаковка / gift опции

### Схема (db:push, нула нови таблици)
`shops`: `giftWrapEnabled boolean NOT NULL DEFAULT false`,
`giftWrapFeeCents integer NOT NULL DEFAULT 0`.
`orders`: `giftWrap boolean NOT NULL DEFAULT false`, `giftNote text NOT NULL
DEFAULT ''`, `giftWrapFeeCents integer NOT NULL DEFAULT 0` (snapshot на таксата).

### Поток
1. **Настройка** в „Плащане и доставка" (`/dashboard/fulfillment`): toggle
   „Предлагам подаръчна опаковка" + такса (PriceInput, 0 = безплатна).
2. **Checkout** (публичен + ръчна поръчка N11): при включено — чекбокс
   „Опаковай като подарък (+X €)" + „Текст за картичка" (опц., ≤200,
   sanitizeText). Таксата се добавя към total НА СЪРВЪРА в
   createOrder/createManualOrder (клиентската сума е преглед); Zod полета в
   orderSchema/manualOrderSchema.
3. **Показване:** ред „Подаръчна опаковка" в сумите (потвърждение, имейли,
   детайл в панела) + текстът за картичката откроен за търговеца в детайла.

## N12 — Връщания (cash вариант; Stripe refund идва с M2/S2)

### Схема (db:push)
- Enum `order_status` + 2 стойности: `return_requested`, `returned`.
- `shops`: `returnWindowDays integer NOT NULL DEFAULT 14` (позволени 14/30/45 —
  валидира се в Zod; настройка в „Плащане и доставка" до N9 опциите).
- `orders`: `returnReason text NOT NULL DEFAULT ''`, `returnRequestedAt
  timestamptz` (за срока + одит).

### Преходи (ALLOWED_TRANSITIONS)
- `completed → return_requested` — САМО от купувача, през публичната страница
  на поръчката (token линка), в срок `shop.returnWindowDays` дни от
  `updatedAt` на завършването* + опционална причина (≤500, sanitizeMultiline).
  Нова публична action `requestReturn(shopSlug, {orderId, token, reason})` —
  rate limit + token проверка (не по orderId сам).
  *Срокът се брои от последната промяна към completed (updatedAt при
  завършване); достатъчно точно без нова колона completedAt.
- `return_requested → returned` — търговецът приема: наличностите се
  ВЪЗСТАНОВЯВАТ (същата логика като cancel — извлича се обща функция
  restoreStock, не се копира).
- `return_requested → completed` — търговецът отказва връщането (обратно към
  „Завършена", без движение по наличности).

### Известия
- Заявка → имейл + push до търговеца („Заявено връщане за #0012" + причината).
- `returned` / отказано връщане → имейл до купувача (разширява се STATUS_EMAIL
  механизмът; отказът праща „поръчката ти остава завършена" вариант).

### Панел + публична страница
- `ORDER_STATUS_LABELS` + двата статуса: „Заявено връщане" (warning), „Върната"
  (danger). Статус филтърът ги включва автоматично (генерира се от labels).
- Детайл на поръчката: при `return_requested` — причината + бутони „Приеми
  връщането" / „Откажи връщането" (ConfirmDialog).
- Публична страница на поръчката: при `completed` в срока — бутон „Заяви
  връщане" + поле причина; след заявка — статус съобщение.

### Приходи
`returned` се изключва отвсякъде, където се изключва `cancelled`:
`getMonthRevenue`, S5 analytics (метрики, дневни, топ продукти) — условието
става `status NOT IN ('cancelled','returned')` (обща константа/helper, не
копирани списъци).

## Извън обхват (YAGNI)
Stripe/картов refund (M2/S2) · частично връщане (само цялата поръчка) ·
снимки/доказателства към заявката · автоматично одобрение · куриерски
етикети за връщане (M3).

## Тестване (потребителят)
- **N9:** включи опаковката с такса 2 € → checkout показва чекбокс + картичка;
  сумата в имейла/панела включва реда; ръчна поръчка също; изключи → изчезва.
- **N12:** завърши поръчка → от token линка „Заяви връщане" с причина →
  търговецът вижда „Заявено връщане" + причината; „Приеми" → „Върната",
  наличностите се връщат, приходите в аналитиката намаляват; „Откажи" →
  обратно „Завършена" + имейл до купувача; поръчка извън срока → бутонът
  липсва; смени срока на 45 дни → бутонът се появява отново.
