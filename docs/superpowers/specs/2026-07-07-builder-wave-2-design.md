# Website Builder — Вълна 2 (дизайн)

Дата: 2026-07-07 · Пътна карта: `docs/superpowers/plans/2026-07-06-builder-roadmap.md`

Вълна 2 дава на търговеца контрол над **съдържание, което сега е заключено**:
правни текстове, навигационно меню, типография, hero видео. Четирите фийчъра
разширяват `siteSettingsSchema` (или `shops`) с нови **незадължителни** полета —
старите магазини получават дефолти автоматично, нищо не се чупи. Всичко минава
през draft/publish потока като досега.

**Одобрени решения (2026-07-07):** правни текстове = override по секция; меню =
авто + ръчни линкове + „Още" dropdown; шрифт = курирани двойки; video hero =
качване на файл, само за poster вариант, reduced-motion → постер.

---

## #1 Правни текстове — override по секция

**Проблем:** `terms/page.tsx` рендерира 5 авто-генерирани секции от
`lib/legal-template.ts` — нередактируеми. Юридически риск за BG (ЗЗП 14 дни,
GDPR).

**Данни:** ново поле в `siteSettingsSchema`:
```
legalOverrides: Record<string, string>  // ключ = section id, стойност = текст
```
Ключове: `general`, `payment`, `delivery`, `returns`, `privacy` (5-те секции).
Празно/липсващо → шаблонът. Всяка стойност max ~5000 знака, sanitizeMultiline.

**Шаблон рефактор:** `legalSections(shop)` → `legalSections(shop, overrides)`.
Всяка секция получава стабилен `id`. За секция с непразен override → текстът се
разбива на параграфи по празен ред; иначе шаблонните параграфи. Дизайнът
(номерирани секции, hairlines) не се пипа.

**Редактор:** нов таб **„Правно"** в `editor.tsx` (масив TABS, до „За нас").
5 Textarea-та, всяко с:
- Label = заглавието на секцията
- Placeholder = шаблонният текст (за да вижда какво замества)
- hint „Празно = стандартният текст"
- (по избор) линк „Върни стандартния текст" = изчиства override-а

**Storefront:** `terms/page.tsx` подава `settings.legalOverrides` на
`legalSections`. Толерантно: непознат ключ се игнорира.

## #3 Навигационно меню — авто + ръчни линкове, „Още" dropdown

**Проблем:** `buildNav` (header/shared.tsx) е напълно авто — категории ако ≤4,
иначе „Продукти" + За нас + Контакти. Търговецът няма контрол.

**Данни:** ново поле в `siteSettingsSchema`:
```
navLinks: Array<{ id: uuid, label: string(max 40), href: string(max 300) }>  // max 8
```
Празно = текущото авто поведение. Непразно = ръчните линкове се **добавят** към
авто-базата (не заменят — решение „авто + ръчни"). Href = вътрешен път
(`/products`) или външен URL (`https://…`).

**buildNav рефактор:** `buildNav(base, rootCategories, navLinks)` → авто-базата +
ръчните. Външните линкове получават флаг `external` (target=_blank).

**Редактор:** в таб „Тема", нова секция „Меню":
- dnd-kit sortable списък на ръчните линкове (същите patterns като SectionsList)
- всеки ред: Input етикет + Input href + премахване
- „+ Добави линк" (до 8)
- празно състояние: „Менюто се сглобява автоматично от категориите ти."

**Storefront — препълване (`NavOverflow`):** нов компонент. Header вариант 1 и 2
рендерират първите N линка инлайн; останалите → dropdown „Още ∨" (CSS/JS,
затваря при клик отвън + Escape, клавиатурно достъпен, темови токени). Праг =
фиксиран (>5 линка → останалите в „Още"); без динамично измерване на ширина
(предсказуемо, без layout трептене). Вариант 3 (минимал) вече крие всичко в
бургер → „Още" не важи. Мобилният бургер показва пълния списък.

## #8 Избор на шрифт — курирани двойки

**Проблем:** шрифтът е заключен в темата (`--sf-font-heading`/`--sf-font-body`
в themes.ts). Няма избор.

**Данни:** ново поле в `siteSettingsSchema`:
```
fontPair: enum(["theme", "editorial", "modern", "warm", "industrial", "clean"])  // default "theme"
```
`"theme"` = шрифтът на избраната тема (текущото). Всички шрифтове са ВЕЧЕ
заредени в root layout през next/font — не се добавят нови.

**Двойки** (`lib/font-pairs.ts`, heading + body от заредените 8):
| id | Име | Heading | Body |
|----|-----|---------|------|
| theme | Тема (по избраната тема) | — | — |
| editorial | Editorial | Playfair | Inter |
| modern | Модерен | Space Grotesk | Inter |
| warm | Топъл | Lora | Inter |
| industrial | Индустриален | Condensed (Oswald) | Inter |
| clean | Изчистен | Onest | Inter |

**themeStyle рефактор:** ако `fontPair !== "theme"`, override-ва
`--sf-font-heading` + `--sf-font-body` от двойката; иначе стойностите на темата.
Един блок логика в `themeStyle` (themes.ts). `--sf-heading-weight` остава от
темата (двойките не пипат тежести).

**Редактор:** в таб „Тема", секция „Шрифт" — картички per двойка, всяка показана
с **реалния шрифт** (визуален preview: „Аа" + примерен текст). Активната =
бранд бордер.

## #9 Video hero — качване на файл

**Проблем:** hero поддържа само снимки (`imagePaths`). Видео е стандарт за
мода/streetwear (теми puls/vitrina).

**Данни:** ново поле в hero `data`:
```
videoPath: string(max 300)  // default ""
```
Празно = снимков hero (текущото). Постер = първата снимка от `imagePaths`
(reduced-motion + докато видеото зарежда).

**Storage:** нов тип в `requestImageUpload` (uploads.ts) + storage.ts:
- `kind: "video"` → папка `shops/{id}/video/`
- `ALLOWED_VIDEO_EXT = ["mp4", "webm"]`
- `MAX_VIDEO_BYTES = 15 * 1024 * 1024`
- `publicVideoUrl()` (или преизползване на publicImageUrl — същият bucket)
- Uploader: параметризиране на съществуващия `ImageUploader` за видео accept
  (преизползва прогрес/XHR логиката) ИЛИ малък отделен — преценя се при
  имплементация; целта е да не се дублира upload прогресът.

**Storefront (hero варианти):** САМО `poster` вариант (пълноекранна снимка-фон).
Ако `videoPath` зададен:
```
<video autoPlay muted loop playsInline poster={firstImage}>
  <source src={videoUrl} />
</video>
```
вместо `<Image>`. `prefers-reduced-motion` → само постерът (без видео) — правило
на проекта, уважава се (CSS `motion-reduce` или media query в компонент).
split/statement вариантите НЕ получават видео (снимка-картичка, не фон).

**Редактор:** в hero section-form — video uploader под снимките, с бележка:
„Видеото замества снимката като фон (само за оформление „плакат"). Първата
снимка е постер (показва се на телефони с пестене на батерия и докато видеото
зарежда)."

**Известно ограничение (iOS moov):** генерираните видеа имат `moov` atom в края
→ на iOS standalone PWA показват само първи кадър (документирано в CLAUDE.md). За
ПОТРЕБИТЕЛСКИ качени видеа не пускаме ffmpeg на сървъра → UI бележка „за
най-добра съвместимост MP4 с faststart". Обикновените телефонни видеа са ОК;
рядък edge. Бъдещо подобрение: сървърен transcode.

**Известно ограничение (autoplay по браузър — 2026-07-06):** hero видеото
разчита на muted autoplay. Chrome го пуска надеждно; **Opera** (и data-saver
режими, Safari Low Power) са по-строги и понякога отказват autoplay → без
постер потребителят вижда празен/тъмен hero. Реализирани смекчения:
- `HeroVideo` (hero-video.tsx) подсилва autoplay с програмен `video.play()` при
  `loadeddata` (по-надежден от HTML `autoplay` сам).
- Постерът е ВИНАГИ под видеото (не само reduced-motion) → ако видеото не тръгне,
  се вижда снимката, не черно.
- Когато НЯМА качена снимка → тонален градиент fallback (пак не черно).
- Preview iframe в редактора има `allow="autoplay"`; въпреки това строги браузъри
  може да покажат само постер там — реалният сайт е меродавен.
**Практическа препоръка към търговеца:** ВИНАГИ качвай и снимка-постер до
видеото — тя покрива всеки браузър/режим, който откаже autoplay. Празният hero
се появява само при видео БЕЗ постер в строг браузър (напр. Opera).
**Future check:** ако се стигне до пълна кросбраузърна гаранция → или сървърен
transcode + „click to play" fallback, или премахване на видео при засечен
autoplay-block (JS проба + swap към постер). Прекалено специфично за сега.

## Ред на имплементация (по тежест)

1. **#1 Правни текстове** — най-важен (юридически), най-прост (textarea override).
2. **#8 Шрифт** — прост (enum + themeStyle override + preview картички).
3. **#3 Меню** — среден (dnd списък + NavOverflow dropdown + buildNav рефактор).
4. **#9 Video hero** — най-тежък (storage тип + uploader + video рендер + постер).

Всеки минава `pnpm check` преди следващия. Гейт накрая + инструкции за тестване.

## Тестване (ръчно, от потребителя)

- Правно: таб „Правно" → override на „Връщане" → сайт /terms показва новия текст;
  изчистване → връща шаблона.
- Шрифт: таб „Тема" → двойка „Editorial" → hero/заглавия сменят шрифта в preview.
- Меню: таб „Тема" → +3 ръчни линка → header показва „Още" dropdown при >5.
- Video: hero (poster вариант) → качи MP4 → фонът става видео; reduced-motion →
  постерът.
