# Спец: Конверсионно трио (търсене в хедъра · доставка на продукта · trust badges на checkout)

**Дата:** 2026-07-11
**Контекст:** Продуктовият gap одит (2026-07-09) сочи три евтини високоефектни storefront
подобрения като „топ-3 в един заход": търсене в site-wide хедъра, доставка/срок на
продуктовата страница, trust badges на checkout. Всичките реюзват съществуващо и са
чист код (нула нови ключове). Един спец → един план → имплементация.

**Дизайн език:** storefront — само `--sf-*` токени (не „Пазарен ден"); mobile-first; един
компонент рендира всички теми.

---

## A. Търсене в site-wide хедъра

**Днес:** пълна търсачка съществува само на `/s/{slug}/products` (GET форма, param `search`,
заявка `getActiveProducts({ search })`). На всяка друга страница (начало, продукт, количка…)
няма вход за търсене → купувачът трябва първо да иде на „Продукти".

**Решение (избор на потребителя): икона → отваряща лента (overlay).**
- Нов client компонент `HeaderSearch` — лупа-икона бутон; клик отваря лента на цялата
  ширина под хедъра с autofocus input. `Enter`/submit → GET към `/s/{slug}/products` с
  `name="search"`. `Esc`, бутон „×" и клик извън лентата затварят.
- **Монтиране:** в action клъстера на 3-те header варианта (до `CartButton`/`FavoritesButton`,
  през общ `shared.tsx` export) + отделен вход „Търсене" в мобилното меню (`MobileMenu` drawer).
- **Реюз:** същият `search` param + `getActiveProducts` — БЕЗ промяна по заявката/`/products`.
- **Данни:** компонентът получава `base` (`/s/{shop.slug}`) от header варианта.
- **Edge:** празно/само-празни символи → не сабмитва (native `required` + trim guard).
- **A11y:** `aria-label` на бутона; `role="search"` на формата; фокусът влиза в input при
  отваряне и се връща на бутона при затваряне; уважава touch target ≥44px.

## B. Доставка/срок на продуктовата страница

**Днес:** `/s/{slug}/p/{productSlug}` не показва нищо за доставка — причина №1 за изоставяне.

**Решение:** компактен блок **„Доставка"** след зоната за количка (`VariantPicker` +
евентуалния `StockAlertForm`), преди „Описание/Характеристики".
- Нов **server** компонент `ProductDelivery` — `getShippingMethods(shop.id)`, филтрира `active`;
  за всеки метод ред: **име** · **цена** (`formatPrice` от центове, или „Безплатна" при `0`) ·
  **срок** (`deliveryHoursLines(m.deliveryHours)` — вече ползван на checkout).
- **Реюз:** `getShippingMethods` (`db/queries/fulfillment.ts`), `formatPrice` (`money.ts`),
  `deliveryHoursLines` (`working-hours.ts`).
- **Стил:** hairline карта с `--sf-*` токени (икона `truck` + списък); дискретна, не крещяща.
- **Edge:** няма активни методи за доставка → блокът НЕ се рендира (нищо празно).

## C. Trust badges на checkout (авто)

**Днес:** `TrustBadgesSection` съществува, но е **builder секция** (ръчно конфигурирана,
по желание на страница) — не е на checkout.

**Решение:** **авто** hairline лента над `CheckoutForm`, деривирана от реални данни (без
конфигурация):
- `shop.returnWindowDays > 0` → „Връщане до N дни" (икона `return`).
- активен payment метод от тип COD → „Плащане при доставка" (икона `truck`).
- ВИНАГИ → „Сигурна поръчка" (икона `shield-check`) + „Без регистрация" (икона `check`).
- Нов компонент `CheckoutTrustBadges` (props: `returnWindowDays`, `hasCod`) → рендира hairline
  лента в стила на `trust-badges` вариант 2 (икона + текст, разделени с точки).
- Чистата деривация се изнася в тествана функция `buildCheckoutBadges(returnWindowDays, hasCod)`
  → `{ icon: IconName; text: string }[]` (винаги ≥2 badge-а).
- **Реюз:** вътрешния `Icon` set, `--sf-*` токени.
- **Данни на checkout страницата:** `shop.returnWindowDays` + `activePayment` (за `hasCod`).
  COD се разпознава по типа на payment метода (сверява се в плана срещу схемата).
- **Позиция:** между `PageHeader` и `CheckoutForm` (виждаш доверието преди да попълваш).

---

## Тестване

- **Unit** (Vitest) за чистата логика: `buildCheckoutBadges` (0 дни връщане → без return badge;
  >0 → с него; hasCod true/false; винаги сигурна+без регистрация; ред/съдържание).
- **Гейт** `pnpm check` (lint + unit + build).
- **Ръчна визуална проверка** от потребителя (light + dark + 375px): търсене от хедъра на
  различни страници + мобилно меню; блок „Доставка" на продукт с/без методи; trust лентата на
  checkout с/без COD и с/без връщане. (Playwright не се ползва за естетика.)

## Не-цели / решения

- **Без live-search/автокомплийт** — само submit → `/products` (YAGNI; трафикът е малък).
- **Без промяна** на съществуващата `/products` търсачка/`getActiveProducts`.
- **Trust badges на checkout са авто** (не конфигурируеми) — конфигурируемата builder секция
  остава отделна и непроменена.
- Доставка на продукта е само инфо (не смята нищо) — реалната цена си остава на checkout през
  `priceCart` (единствен ценови източник).
