# Storefront секции — тотален редизайн (дизайн спец)

**Дата:** 2026-07-05
**Статус:** чака одобрение от собственика
**Обхват:** всичките 13 storefront секции + продуктова карта + hero + секционна обвивка
**База:** визуален одит на Ателие секция-по-секция (Playwright, 1440×900) +
`docs/research/2026-07-05-ecommerce-design-research.md` + недоволство на собственика
от текущия „плосък, безопасен" вид.

## Изисквания от собственика (2026-07-05)

1. **Адаптивност към съдържанието** — независимо колко текст/продукти/медия има,
   дизайнът работи перфектно (1 продукт, 50 продукта, 2 категории, 10 категории).
2. **Не safe** — комплексни, красиви, впечатляващи секции.
3. **Design science** — всяка от 9-те теми ползва правилните си цветове и дизайн
   логика; редизайнът чете само `--sf-*` (никакъв хардкод).
4. **Оптимален вид** — конкретно: hero скучен и не покрива екрана; 5 продукта се
   чупят (искане: ≤4 → един ред; 5+ → карусел, пасващ на темата); категориите
   изглеждат зле при 2 (искане: различен дизайн според броя).

## Диагноза от одита (защо изглежда плоско)

| # | Проблем | Засегнати секции |
|---|---|---|
| Д1 | Всичко е на един фон (`--sf-bg`) — няма ритъм, секциите се сливат | всички |
| Д2 | Една композиция навсякъде: „заглавие вляво + грид под него" | всички |
| Д3 | Hero не използва пространството (малка карта-снимка, много празно) | hero |
| Д4 | Броят елементи не е дизайн-случай (2 категории/2 отзива/5 продукта висят) | featured, category, testimonials |
| Д5 | Категориите са голи текстови кутии — нула визуална покана | category-grid |
| Д6 | Продуктовата карта е плаха: малка снимка, тънка рамка, никакъв hover живот | product-card (навсякъде) |
| Д7 | `text-white` върху `--sf-primary` — нечетимо при светъл primary (Пулс) | hero CTA, promo, checkout, cart |
| Д8 | Тъмните теми разчитат на сенки, които не се виждат на тъмен фон | всички карти |
| Д9 | Секциите не водят никъде — няма „Виж всички →", няма CTA ритъм | featured, gallery, testimonials |

## Фундамент (Фаза 0) — прави всичко останало възможно

### 0.1 Нови теми токени (в `ThemeVars`, всяка от 9-те теми получава стойности)

| Токен | Роля | Пример (Пулс) |
|---|---|---|
| `--sf-on-primary` | текст върху плътен primary (решава Д7) | `#111111` (върху неоново жълто) |
| `--sf-surface-raised` | „повдигната" повърхност за карти върху surface фон (Д1/Д8) | `#222222` |
| `--sf-shadow` | сянка на картите per тема — тъмните дават `0 0 0 1px border` вместо сянка (Д8) | без сянка, бордер |
| `--sf-overlay` | градиент/затъмнение върху hero/категорийни снимки за четим текст | `rgba(0,0,0,.55)` |

Всички компоненти спират да пишат `text-white` върху primary → `text-(--sf-on-primary)`.

### 0.2 Секционна обвивка (SectionShell) — editorial заглавен блок

```
[kicker — малък uppercase, letterspaced, в --sf-primary]
[заглавие — clamp(1.75rem → 2.75rem), темов шрифт]
[опц. интро изречение — muted, max-w-prose]        [опц. action: „Виж всички →"]
```

- Всяка секция подава своя kicker (напр. featured → „Магазин", testimonials →
  „Отзиви", gallery → „Галерия"). Дава ритъм и йерархия (Д2).
- `tone` prop: `default` (на `--sf-bg`) | `surface` (пълноширок `--sf-surface` фон) |
  `primary` (плътен брандов момент). Редуването на тонове решава Д1 —
  **рендерът задава автоматично редуване**: секциите се редуват bg → surface, като
  promo/CTA секциите взимат `primary`.

### 0.3 Продуктова карта — редизайн (важи навсякъде)

- Снимка 4:5 (по-висока, image-first — research: „снимката е героят").
- Hover: снимката леко zoom-ва (scale 1.04, 400ms), картата се повдига
  (`--sf-shadow`); втора снимка при hover, ако продуктът има ≥2 (desktop).
- Цена: по-едра, промо → стара цена зачертана + процент badge в `--sf-accent`.
- Тъмни теми: фин светъл бордер вместо сянка (`--sf-shadow` го дава per тема).
- „Изчерпано" → опций overlay етикет, не изчезнала карта.

## Секция по секция (Фази 1–5)

### Фаза 1 — Hero (най-голямо въздействие, Д3)

**`full` (по подразбиране):** 88vh full-bleed. Снимката покрива всичко,
`--sf-overlay` градиент отдолу. Типография: clamp(2.75rem, 7vw, 5.5rem), темов
display шрифт. Подзаглавие muted-light. Двоен CTA: primary бутон
(`--sf-primary` + `--sf-on-primary`) + ghost link „Научи повече ↓". Фин scroll cue
(chevron, respect reduced-motion). Без снимка → типографски hero върху тонален
градиент от `--sf-surface` → `--sf-bg` с едра типография — пак драматичен.

**`split`:** 80vh. Снимката edge-to-edge дясна половина (без заоблена карта,
без padding около нея). Текстът вертикално центриран в лявата, kicker + голямо
заглавие + CTA двойка.

**`slideshow`:** като `full` + crossfade между снимките (5s, пауза при
reduced-motion, точки-индикатори долу).

### Фаза 2 — Featured products (Д4, искане №2)

Адаптивна стратегия по брой продукти:

| Брой | Layout |
|---|---|
| 1 | **Spotlight**: голяма снимка 55% + панел име/цена/описание/CTA „Виж продукта" (мини product-page момент) |
| 2–4 | **Един ред**, grid-cols-{N} — картите растат да запълнят реда, никога втори ред |
| 5+ | **Карусел**: CSS scroll-snap, 4 видими + peek на 5-ата (подсказва скрол), стрелки ‹ › (client комп., уважава reduced-motion), без библиотека |

Заглавен блок с action „Виж всички →" към /products (Д9).

### Фаза 3 — Категории (Д5, искане №3)

Категорийната карта става **визуална**: фонова снимка (първата снимка на най-новия
продукт в категорията — нова лека query) + `--sf-overlay` + име + „N продукта".
Без продукти/снимка → тонален фон (`--sf-surface-raised`) + едра инициал-буква.

Адаптивни layout-и по брой:

| Брой | Layout |
|---|---|
| 1 | пълноширок банер (h-56), име + CTA |
| 2 | две половини 50/50 (h-72) — „collection banners" (research: мода) |
| 3 | триптих 3×(h-64) |
| 4–6 | грид 3 колони × h-48 |
| 7+ | компактен грид 4 колони × h-36 или chips лента |

### Фаза 4 — Социално доказателство и промо

**Отзиви (Д4):** 1 → голям центриран pull-quote (декоративна „ в `--sf-primary`,
clamp типография); 2 → два широки 50/50; 3+ → грид 3 колони; 7+ → карусел като
featured. Инициал-аватар кръгче (от името) + име.

**Trust badges:** от плоска линия → плочки на `--sf-surface-raised` с икона в
кръгче (`--sf-primary` на 10% фон); 2×2 на мобилно, до 6 в ред desktop. При
`tone=surface` секцията се слива с ритъма.

**Промо банер:** min-h-80, едра clamp типография, CTA бутон задължително видим
(`--sf-on-primary`), `--sf-overlay` върху снимката за четимост; без снимка →
плътен `--sf-primary` фон + `--sf-on-primary` текст (брандов момент, tone=primary).

### Фаза 5 — Останалите

- **Image-text:** edge-to-edge снимка 50% (без карта-рамка), kicker + заглавие +
  текст + опц. CTA; редуване ляво/дясно вече е в data.
- **Галерия:** masonry (CSS columns, разновисоки) + hover scale + lightbox
  (`<dialog>`, без библиотека). 1–2 снимки → централен дует, не самотни квадрати.
- **FAQ:** центрирана колона max-w-3xl, заглавието центрирано, по-щедри отстояния,
  chevron анимация.
- **Контакти:** инфото в карта върху `--sf-surface-raised` (икони в кръгчета),
  картата пълни останалото; мобилно — стек.
- **Социални:** кръгли икон-бутони с hover запълване в `--sf-primary`; влиза
  визуално към footer ритъма.
- **Announcement:** остава `--sf-text`/`--sf-bg` (вече оправен контраст), може
  опционален line-clamp.
- **За нас/rich-text:** запазва центрирания editorial вид (вече направен).

## Отношение към section-variants спека (2026-07-05)

Редизайнът предефинира какво е **primary** вариантът на всяка секция.
Адаптивността по брой е **вградено поведение**, не вариант. Вариантите (alt) идват
СЛЕД редизайна и стъпват на новите компоненти (напр. featured alt = карусел
винаги; gallery alt = равна решетка). Спекът за варианти остава валиден, но се
изпълнява след този.

## Технически бележки

- Никакви нови зависимости — карусел/lightbox/masonry са CSS-first + минимален JS.
- Всичко чете само `--sf-*`; нула хардкоднати цветове (изискване №3).
- Нови клиент-компоненти: Carousel (стрелки+snap), Lightbox — в
  `src/components/storefront/`, преизползваеми от секциите.
- Категорийната снимка: 1 доп. заявка в storefront home (grouped по категория,
  кеширана с ISR — без N+1).
- `prefers-reduced-motion` уважен навсякъде (без auto-slideshow/zoom).
- Визуален pass след всяка фаза: Ателие (светла) + Пулс (тъмна) минимум; финално
  всичките 9 × 375px + desktop.
