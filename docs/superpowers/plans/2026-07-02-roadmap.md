# Frizmo Shops — Roadmap на имплементацията

**Спецификация:** `docs/superpowers/specs/2026-07-02-frizmo-shops-mvp-design.md`

MVP-то се изгражда чрез 6 последователни плана. Всеки план завършва с **работещ, тестван, комитнат софтуер без TODO-та** — следващият план стъпва върху реалния код. Детайлният план за всяка фаза се пише при стартирането ѝ (Plan 1 е готов: `2026-07-02-plan-1-foundation.md`).

| # | План | Обхват (секции от спеца) | Definition of Done |
|---|---|---|---|
| 1 | **Фундамент** | §3, §7, §8, части от §5, §12 | Проектът е scaffold-нат (Next.js 15, TS strict, Tailwind 4 с design tokens); UI примитиви (Button, Input, Card, Badge, Spinner); Drizzle + Supabase с таблици `profiles`/`shops`; utility слой (money EUR, slug, sanitize) с unit тестове; регистрация/вход/изход работят end-to-end (Playwright); dashboard shell; `pnpm check` гейт минава. |
| 2 | **Магазин и продукти** | §5 (продукти), §13, dashboard табове Store/Products/Categories | Onboarding wizard (магазин → първи продукт); CRUD за магазин, категории (+подкатегории), продукти с характеристики, варианти и снимки (Supabase Storage, подписани URL-и); `authorizedAction()` с проверка на собственост; unit + e2e тестове. |
| 3 | **Публичен магазин** | §4 (storefront路 routes), §6 | `/s/{slug}` с ISR: начало (секционна система — всичките 13 секции), продукти, продукт, за нас, контакти, условия (шаблонни); 3 теми чрез CSS variables; таб „Уебсайт" с drag & drop редактор + iframe preview; публикуване draft → published; e2e тест. |
| 4 | **Количка и поръчки** | §9, dashboard таб Orders | Количка per-магазин (localStorage); гост checkout с пълна сървърна валидация и транзакция; всички edge cases от §9; имейли (Resend); управление на поръчки със статуси; unit тестове за order транзакцията + e2e гост поръчка. |
| 5 | **Каталог, landing, блог, SEO** | §4 (каталог), §11 | `/shops` и `/products` с филтри + full-text търсене (Postgres); landing page (7-те секции от §11); MDX блог с 2 статии; sitemap/robots/OG/JSON-LD; 3 демо магазина с попълнено съдържание. |
| 6 | **Абонаменти и платформен админ** | §10, /admin, §14 | Stripe Checkout + webhooks; lifecycle trial→active→grace→suspended + дневен reconciliation cron; `checkPlanLimit()` приложен навсякъде; /admin (списък магазини, скриване/блокиране, статистика); launch checklist по §14. |

**Ред на изпълнение:** строго 1 → 6. Спецификацията е източникът на истината; при отклонение по време на имплементация се обновява спецът (с ADR запис в `docs/decisions/`), не се импровизира мълчаливо.

**Компонентна библиотека:** примитивите от §7 се добавят в плана, в който за първи път потрябват (YAGNI) — напр. SlidePanel/Table в План 2, Accordion в План 3. Всеки нов примитив влиза в `components/ui/` с типизирани props и всички състояния.
