# Bulgarian Language Guide — UI текстове на Frizmo Shops

> Практически справочник за писане на КОРЕКТЕН и естествено звучащ български в
> UI-я. Чете се преди писане/редакция на всеки потребителски текст (toast-ове,
> бутони, съобщения за грешки, етикети, имейли). Целта: без граматически грешки
> и без „преведено от английски" звучене.

---

## 1. Числово съгласуване (най-честата грешка)

Съществителното след число се съгласува по число. Проверявай ВИНАГИ при
динамични съобщения („N продукта", „N промоции").

### Правило
- **1** → единствено число: `1 продукт`, `1 промоция`, `1 поръчка`, `1 ревю`.
- **2, 3, 4, … (всяко друго число)** → множествено число: `5 продукта`,
  `3 промоции`, `12 поръчки`, `7 ревюта`.
- **0** → множествено число: `0 продукта`, `0 поръчки`.

### Съгласуване и на ПРИЧАСТИЕТО/ПРИЛАГАТЕЛНОТО (не само на съществителното!)
Ако преди числото има причастие (напр. „Премахнат/а/и"), то СЪЩО се съгласува:
- ✗ `Премахнати 1 промоция` (грешно — „премахнати" е мн.ч., „1 промоция" ед.ч.)
- ✓ `Премахната 1 промоция` (ед.ч. + ед.ч.)
- ✓ `Премахнати 3 промоции` (мн.ч. + мн.ч.)

Родът на причастието следва рода на съществителното:
- промоция (ж.р.) → Премахна**та** 1 · Премахна**ти** 3
- продукт (м.р.) → Изтри**т** 1 · Изтри**ти** 5
- ревю (ср.р.) → Изтри**то** 1 · Изтри**ти** 4

### Как да го пишем в код
Прави хелпър за pluralization вместо ръчно тернарно навсякъде, ИЛИ преформулирай
изречението да избегне съгласуването (виж §5).

```ts
// Просто съгласуване на съществителното:
const productWord = (n: number) => (n === 1 ? "продукт" : "продукта");
`${n} ${productWord(n)}`; // "1 продукт" / "5 продукта"
```

За причастие + съществително по-добре преформулирай (по-долу), защото трябва да
съгласуваш ДВЕ думи.

## 2. „Мъжка бройна форма" (специален капан)

За МЪЖКИ РОД, неодушевени предмети, след число се ползва **бройна форма** на
`-а/-я`, а НЕ обикновеното мн.ч. на `-и`:
- ✓ `3 продукта` (бройна форма), НЕ `3 продукти`
- ✓ `5 лева`, НЕ `5 левове`
- ✓ `2 града`, НЕ `2 градове`
- ✓ `4 метода`, НЕ `4 методи`

НО за ОДУШЕВЕНИ (хора) се ползва обикновено мн.ч.:
- ✓ `3 клиенти` / по-добре `3-ма клиенти` (с бройно числително), НЕ `3 клиента`
- ✓ `2-ма търговци`

За женски и среден род НЯМА бройна форма — просто мн.ч.:
- `3 поръчки`, `5 промоции`, `4 ревюта`, `2 категории`.

Списък с думи от проекта (единствено · множествено/бройно):
| дума | 1 | 2+ |
|------|---|-----|
| продукт (м) | продукт | продукт**а** |
| поръчка (ж) | поръчка | поръчк**и** |
| промоция (ж) | промоция | промоци**и** |
| ревю (ср) | ревю | ревю**та** |
| абонат (м, одуш.) | абонат | абонат**и** |
| категория (ж) | категория | категори**и** |
| артикул (м) | артикул | артикул**а** |
| ден (м) | ден | дни (неправилно мн.ч.) |
| лев/евро | 1 евро | 5 евро (евро е неизменяемо!) |
| получател (м, одуш.) | получател | получател**и** |

## 3. Членуване (определителен член)

Пълен член `-ът/-ят` за м.р. подлог; кратък `-а/-я` за м.р. допълнение.
Женски → `-та`; среден → `-то`; множествено → `-те/-та`.

Практично правило за UI (кратките текстове рядко имат сложен синтаксис):
- Подлог (върши действието) → пълен член: „**Продуктът** е запазен."
- Допълнение (върху него се действа) → кратък член: „Виж **продукта**."
- При съмнение: ако можеш да замениш с „той" → пълен член; ако с „него" → кратък.

Не членувай след число или предлог там, където не трябва:
- ✓ `3 продукта` (без член)
- ✓ `виж всички продукти` (без член — неопределено множество)
- ✓ `Продуктите с нисък склад` (членувано — конкретни, определени)

## 4. Типографски кавички и тире (правило на проекта)

- Кавички: **„…"** (U+201E отваряща + U+201C затваряща), НИКОГА прав `"`.
  Прав `"` в BG текст чупи JS стрингове/lint и е типографски грешен.
- Тире в диапазон: **–** (en dash), не `-`: „от 20 – до 40".
- Многоточие: … (един знак), не три точки.
- Дълго тире за пояснение: — (em dash) — както в този ред.

⚠️ В YAML frontmatter (блог): затваряща `"` (прав) чупи стойността. Ползвай
пълна двойка „…" или преформулирай без кавички.

## 5. Тон и стил — да не звучи „преведено"

### Обръщение: на „ти", не на „Вие"
Проектът е приятелски към малкия търговец. Второ лице ед.ч.:
- ✓ „Запази промените", „Виж поръчката", „Ще получиш имейл"
- ✗ „Запазете промените", „Ще получите имейл" (твърде формално)

### Активен, кратък изказ
- ✓ „Поръчката е приета"
- ✗ „Вашата поръчка беше успешно приета от системата" (шумно, англицизъм)

### Избягвай англицизми и буквални калки
- ✗ „Успешно запазено" → ✓ „Запазено"
- ✗ „Моля, изберете опция" → ✓ „Избери…"
- ✗ „Невалиден вход" → ✓ „Невалидна стойност" / „Провери полето"

### Преформулирай, за да избегнеш трудно съгласуване
Вместо да се бориш с „Премахнат/а/и N промоция/и", често е по-чисто:
- „Промоции, паднали под новата цена: N"
- „N без промоция вече" — ако пасва
- Или две отделни съобщения за ед./мн. през тернар.

### Съобщения за грешки — какво + как да се оправи
- ✗ „Грешка."
- ✓ „Цената трябва да е по-голяма от 0."
- ✓ „Промо кодът вече не е валиден. Премахни го и опитай пак."

### Бутони — глагол в повелително или съществително
- ✓ „Запази", „Изтрий", „Приложи", „Нов продукт", „Импорт CSV"
- Кратко, без точка накрая.

## 6. Чести грешки — бърз чеклист преди commit на текст

- [ ] Число + съществително съгласувано? (1 → ед.ч., 2+ → мн.ч./бройна форма)
- [ ] Причастие/прилагателно преди числото СЪЩО съгласувано по число и род?
- [ ] Мъжка бройна форма `-а/-я` за неодушевени (3 продукт**а**, не продукт**и**)?
- [ ] Типографски кавички „…", не прав `"`?
- [ ] На „ти", не на „Вие"?
- [ ] Няма ли англицизъм/буквален превод („успешно", „моля изберете")?
- [ ] Съобщението за грешка казва ли КАК да се оправи?
- [ ] Няма невидими контролни символи (генерацията понякога ги вкарва)?

## 7. Референтни примери от проекта (правилни)

- `Готово — премахната 1 промоция` / `Готово — премахнати 3 промоции`
- `5 продукта · 2 категории`
- `Изпратено до 42 абонати` (одуш. → мн.ч. на -и)
- `Срокът за връщане (14 дни) е изтекъл`
- `Поръчката ти е потвърдена`
- `Няма продукти по тези филтри. Опитай с други стойности.`
