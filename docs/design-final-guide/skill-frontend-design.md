> ПРЕДИ имплементация на промени по дизайна — платформен UI или storefront.

## Какво е

Официалният дизайн skill на Anthropic — **№1 по инсталации** в екосистемата
(~626K към 07.2026, Snyk го класира първи). Инсталиран като плъгин
`frontend-design@claude-plugins-official`; вика се чрез
`Skill: frontend-design:frontend-design`.

**Роля:** дава ЕСТЕТИЧЕСКИ ХАРАКТЕР — маха генеричния „AI slop" вид.
Не е база данни (за данни виж ui-ux-pro-max) — той е дисциплина на мислене.

## Какво налага (същината)

1. **Преди код — ясна естетическа посока.** Избира се ЕДНА крайност и се
   изпълнява прецизно: brutally minimal / editorial / luxury / organic /
   industrial / playful… Интенционалност, не интензивност.
2. **„Какво го прави незабравим?"** — една запомняща се идея per дизайн.
3. **Типография:** отличителни шрифтове; display + body двойка.
   ЗАБРАНЕНИ: Inter/Roboto/Arial/system като естетически избор; и
   конвергенция към „модните" (напр. Space Grotesk навсякъде).
4. **Цвят:** доминантен цвят + остри акценти > плахи равни палитри.
   Винаги CSS variables.
5. **Движение:** високо-въздействени моменти (оркестриран page load със
   staggered reveals) > разпръснати микро-анимации. CSS-first.
6. **Композиция:** асиметрия, застъпване, чупене на грида, диагонален поток,
   щедро negative space ИЛИ контролирана плътност.
7. **Дълбочина:** текстури/градиент мешове/noise/зърно вместо плоски фонове.
8. **ЗАБРАНЕНО:** лилави градиенти на бяло, предвидими шаблонни layouts,
   cookie-cutter компоненти без контекстен характер.

## Как се прилага В ТОЗИ проект (важните уговорки)

- **Storefront (/s/{slug}):** посоката се задава ОТ ТЕМАТА (9-те теми са
  „естетическите крайности": Оникс = luxury, Пулс = смела улична, Ателие =
  занаятчийска…). Skill-ът се ползва за КАЧЕСТВОТО на изпълнение във вече
  избраната посока — НЕ за да сменя шрифтове/цветове на тема (те живеят
  само в `src/lib/themes.ts` като `--sf-*`). Никакви хардкоднати стойности.
- **Платформен UI (landing/dashboard):** посоката е „Пазарен ден"
  (спец 2026-07-03) — skill-ът подсилва editorial-а ѝ, не я подменя.
- Забраната за Inter НЕ отменя тема, която съзнателно ползва Inter като
  body (Класик/Витрина) — темовият избор е решение, не дефолт.
- Малките клишета, които този skill лови: равни safe grid-ове навсякъде,
  еднакви H2, плоски секции без ритъм, липса на „незабравим" момент.

## Кога да се вика

- Нова секция/страница/компонент с видима повърхност.
- Редизайн или „не изглежда достатъчно добре".
- НЕ за: чисто поведенчески/бекенд промени.

## Работен ред (заедно с ui-ux-pro-max)

1. Прочети този документ + `skill-ui-ux-pro-max.md`.
2. `frontend-design` → определи/потвърди естетическата посока и
   „незабравимия" момент на промяната.
3. `ui-ux-pro-max` → провери данните (UX правила, палитри, типография).
4. Имплементирай само през токени (`--sf-*` / tokens.css).
5. Провери light+dark теми + 375px; `pnpm check` преди commit.
