# Frizmo Shops — Визуален редизайн: пълна дизайн спецификация

**Дата:** 2026-07-03
**Статус:** Ревизиран след критичен преглед — изпълнява се на фази (§20), НЕ като един план; всяка фаза с индивидуално одобрение
**Обхват:** Замества визуалния слой на цялата платформа (landing, каталог, storefront теми, dashboard, checkout). Архитектурата НЕ се пипа — променят се стойности в tokens.css, THEME_PRESETS, шрифтове и се полират компоненти. Добавя се motion слой (Motion) и Radix основа под интерактивните примитиви.
**Позициониране на амбицията:** не „приличен сайт", а сайт, който при първо отваряне казва „това е сериозен, скъп продукт" — и го казва тихо.

---

## 0. Дизайн теза (една мисъл, която управлява всяко решение)

> **„Тих премиум с жив продукт."**
> Платформата изглежда като скъпо ателие, не като SaaS шаблон: топла хартиена основа, дълбоко нефритено зелено, едра уверена типография с кирилица — и **един** зрелищен елемент: живата мини-витрина в hero-то, която се сглобява пред очите на посетителя. Всичко останало е дисциплинирано, бързо и безшумно.

Правило за всяко бъдещо решение: ако елемент не обслужва тезата — маха се. Смелостта се харчи на едно място (hero витрината); всичко около нея е прецизност, не декорация.

**Какво съзнателно НЕ правим (антишаблон):** ярки градиенти по бутони; glassmorphism; неонови акценти; черен фон с киселинно зелено; крем + теракота + serif (клишето на AI-генерирания дизайн 2025–2026); повече от една CTA в hero; тежки видеа; паралакс заради паралакса; бели `#FFFFFF` плоскости на големи площи.

## 1. Цветова система

### 1.1 Философия
- **Основата е „хартия", не „екран":** топло off-white, което прави интерфейса спокоен и премиум вместо клиничен.
- **Един бранд цвят** (дълбоко нефритено зелено) — доверие, стабилност, „местно и органично" — точно нишите храни / ръчна изработка / мода.
- **Един топъл акцент** (кехлибар) — САМО за промоции, badges и малки сигнали. Никога за primary CTA (CTA е винаги бранд зеленото — така окото се учи: зелено = действие).
- Неутралната скала е **топла** (stone-базирана, с лек зелен подтон в тъмните стойности) — това е разликата между „шаблон" и „бранд".

### 1.2 Light палитра (пълни стойности за tokens.css)

### 1.4 Сенки и повърхности
- Сенките са **шепот**: `--shadow-card: 0 1px 2px rgb(28 36 32 / 0.05), 0 4px 12px rgb(28 36 32 / 0.05)`; `--shadow-pop: 0 4px 8px rgb(28 36 32 / 0.06), 0 16px 32px rgb(28 36 32 / 0.10)` (модали, dropdown-и).
- Основният носител на дълбочина е **`1px` граница + фонова разлика** (`bg` → `bg-raised`), не сянка. Карта = raised фон + line граница + shadow-card. Никакви твърди drop shadows.
- Hover на кликаема карта: границата става `line-strong` + сянката се усилва + `translateY(-2px)` — трите заедно, 200ms.

### 1.5 Радиуси
`--radius-sm: 8px` (inputs, badges) · `--radius-md: 12px` (бутони, малки карти) · `--radius-lg: 16px` (карти, модали) · `--radius-xl: 24px` (hero mockup, hero изображения) · `--radius-full` (avatar, pill). Консистентността тук е 50% от „скъпия" вид — нула ad-hoc стойности.

## 2. Типография

### 2.1 Критично ограничение: кирилица
Магазините и купувачите са на български. **Всеки шрифт в системата задължително има пълна кирилска поддръжка.** Space Grotesk (сегашната `modern` тема) НЯМА кирилица — подменя се.

### 2.2 Шрифтови роли (платформата)

| Роля | Шрифт | Защо |
|---|---|---|
| Display / заглавия | **Onest** (700/800) | Съвременен геометричен grotesque с характер в едрите размери, отлична кирилица, безплатен (Google Fonts), направен с мисъл за кирилски пазари |
| Body / UI | **Inter** (400/500/600) | Безспорен за UI четимост, пълна кирилица, вече е в проекта |
| Числа/цени/таблици | Inter с `font-feature-settings: "tnum"` | Табличните цифри подравняват цените в списъци и статистики — малкият детайл, който отличава професионален админ |

Зареждане: `next/font` с `subsets: ['cyrillic', 'latin']`, `display: 'swap'`, self-hosted (нула CLS, нула външни заявки).

### 2.3 Типографска скала (major third, с clamp за fluid размери)

| Токен | Стойност | Употреба |
|---|---|---|
| `--text-hero` | `clamp(2.5rem, 6vw, 4.5rem)` / lh 1.05 / tracking -0.03em / Onest 800 | Landing hero |
| `--text-h1` | `clamp(2rem, 4vw, 3rem)` / lh 1.1 / -0.02em / Onest 700 | Заглавия на страници |
| `--text-h2` | `clamp(1.5rem, 3vw, 2rem)` / lh 1.15 / -0.015em / Onest 700 | Секционни заглавия |
| `--text-h3` | `1.25rem` / lh 1.3 / Onest 600 | Карти, подсекции |
| `--text-body` | `1rem` / lh 1.6 / Inter 400 | Основен текст |
| `--text-sm` | `0.875rem` / lh 1.5 | UI, метаданни |
| `--text-eyebrow` | `0.8125rem` / uppercase / tracking +0.08em / Inter 600 / цвят brand | Етикет над секционни заглавия |

Правила: `text-balance` на всички заглавия; `text-pretty` на параграфи; max-width на текстови блокове `65ch`; заглавие + eyebrow + подзаглавие е стандартният секционен ритъм на landing-а.

### 2.4 Шрифтове на трите storefront теми (§6.3 от спеца)

| Тема | Heading | Body | Характер |
|---|---|---|---|
| `classic` | Inter 700 | Inter | Неутрална, безопасна, „универсален магазин" |
| `modern` | **Onest 800** (заменя Space Grotesk) | Inter | Едра, уверена, tech/мода |
| `warm` | **Lora 600** (serif, има кирилица) | **Manrope** | Топла, занаятчийска — храни, ръчна изработка |

Темите се различават ощутимо и по: радиуси (classic 12px / modern 6px / warm 16px), сенки (classic меки / modern плоски с граници / warm по-меки и топли), и плътност на секционните paddings. Разликата между темите трябва да се вижда от 3 метра — не „три нюанса на едно и също".

## 3. Motion система (нов слой)

### 3.1 Библиотека и принципи
- **Motion** (`motion`, бивш Framer Motion) — декларативен, идеален за App Router; покрива всичко нужно. **GSAP не се добавя** — нямаме scroll-storytelling нужди, а всяка излишна зависимост е шум.
- Микроанимациите добавят **смисъл** (ориентация, потвърждение, причинно-следствие), не спектакъл.
- Анимира се само `transform` и `opacity` (GPU композиция, нула layout thrashing).
- **`useReducedMotion` навсякъде**: при reduced-motion всички reveal-и стават мигновени, hero витрината се показва статично сглобена. Това е и WCAG 2.3.3 изискване.

### 3.2 Motion токени (в tokens.css + огледална константа `lib/motion.ts`)

| Токен | Стойност | Употреба |
|---|---|---|
| `--dur-fast` | `150ms` | Hover, focus, малки състояния |
| `--dur-base` | `250ms` | Reveal, модали, dropdown |
| `--dur-slow` | `400ms` | Page transitions, hero последователност |
| `--ease-out` | `cubic-bezier(0.16, 1, 0.3, 1)` | Влизащи елементи (expo-out — бърз старт, меко кацане) |
| `--ease-in-out` | `cubic-bezier(0.65, 0, 0.35, 1)` | Преместващи се елементи |
| Spring (Motion) | `{ type: 'spring', stiffness: 380, damping: 30 }` | Количка badge, toggle-и, drag |

### 3.3 Речник на движенията (изчерпателен — нищо извън него)

| Патърн | Имплементация | Къде |
|---|---|---|
| **Section reveal** | `whileInView`: opacity 0→1, y 24→0, `--dur-base`, `viewport={{ once: true, margin: '-80px' }}` | Всички landing и storefront секции |
| **Stagger grid** | Родителски `staggerChildren: 0.06` върху reveal-а | Продуктови grid-ове, feature карти, каталог |
| **Card hover** | CSS: `translateY(-2px)` + сянка + граница, 200ms | Всички кликаеми карти |
| **Image zoom** | CSS: `scale(1.04)` на снимката в overflow-hidden контейнер, 400ms | Продуктови карти, галерия |
| **CTA hover** | Фон потъмнява + `translateY(-1px)`; активно: `scale(0.98)` | Всички бутони (CSS, не JS) |
| **Modal / SlidePanel** | `AnimatePresence`: overlay fade + панел y/x transform със spring | Модали, mobile bottom-sheet, drawer количка |
| **Cart badge** | Spring `scale: [1, 1.35, 1]` при промяна на брой | Storefront header |
| **Add-to-cart** | Бутонът минава през състояние ✓ „Добавено" (icon swap + brand-soft фон, 1.2s) | Продуктова страница |
| **Toast** | Slide-in отдолу + fade, spring | Sonner конфигурация |
| **Variant снимки** | `AnimatePresence mode='wait'`: crossfade 200ms при смяна на вариант | Продуктова страница |
| **Number tick** | Digits анимация при промяна на сума | Checkout totals, dashboard статистика |
| **Skeleton** | Пулс от `bg-sunken` към `line`, 1.5s | Всички loading.tsx |
| **Page transition (dashboard)** | `template.tsx`: fade + y 8px, 300ms | Между dashboard табове |
| **Accordion** | Височина + fade на съдържанието, `--dur-base` | FAQ, категориен филтър |

**Правило за плътност:** максимум ЕДНА orchestrated анимация на viewport (hero последователността); всичко останало е еднократен reveal или hover. Ако страницата „танцува" — прекалили сме.

## 4. Сигнатурният елемент: живата hero витрина

Единственият спектакъл на сайта. Замества планирания статичен CSS browser-mockup.

**Какво е:** стилизиран browser прозорец (radius-xl, shadow-pop, адресна лента с `frizmo.shop/s/atelie-ruchichka`), в който при зареждане се **сглобява истински мини-магазин**:

1. `0.0s` — прозорецът се появява (fade + y, `--dur-slow`)
2. `0.3s` — header на магазина с лого и навигация се плъзга отгоре
3. `0.5s` — hero снимка на магазина се разкрива (clip-path reveal)
4. `0.7s` — 3 продуктови карти влизат stagger-нато (реални продукти от демо магазините — истински снимки, истински цени в €)
5. `1.4s` — курсор-точка се плъзга към „Добави" на една карта, натиска я, количката в mini-header-а получава badge с spring `scale` — **демонстрира продукта, без да казва дума**
6. `1.8s` — покой; на всеки ~6s курсорът сменя активната тема на мини-магазина (classic → warm → modern), при което цветовете и радиусите на витрината плавно се преливат — това показва И персонализацията, ключовото продуктово обещание

**Имплементация:** чист React + Motion + токени (никакви скрийншоти/видео — не остарява, ~0 KB media, винаги в бранд цветовете). Компонент `marketing/hero-storefront-demo.tsx`, декомпозиран на мини-компоненти. При `prefers-reduced-motion`: показва се финалният кадър статично. На mobile: същата витрина, опростена до 2 продуктови карти, без курсор последователността (само сглобяването).

Вторична употреба: същите мини-UI компоненти (карта „добави продукт", theme picker, поръчка с push нотификация) изграждат визуалите на секцията „Как работи" и на трите „Функции" блока — консистентен визуален език вместо стокови илюстрации.

## 5. Насоки per повърхност

### 5.1 Landing
- **Hero:** eyebrow („Онлайн магазин за малкия бизнес") → hero заглавие → едно изречение подзаглавие → **една** CTA „Създай магазина си безплатно" + тих текстов линк „Разгледай демо магазин →". Вдясно/отдолу: живата витрина. Под fold-линията: лента с 3 сухи доказателства („Без карта за trial · Поръчки с наложен платеж · Готов за 15 минути").
- **Секционен ритъм:** редуване `bg` / `bg-sunken` фонове; вертикален padding `clamp(4rem, 10vw, 7rem)`; всяка секция с eyebrow + h2 + max-65ch увод.
- **Болката:** 3 карти с line-art икони (виж §7) — без клипарт, без емоджита.
- **Цени:** Pro колоната с brand граница 2px + малък badge „Препоръчан"; таблицата с tnum цифри; toggle месечно/годишно не се добавя (нямаме годишни планове в MVP — не лъжем с UI).
- **FAQ:** акордеон с motion; JSON-LD `FAQPage` добавен към съществуващия `SoftwareApplication`.
- **Финална CTA:** плътен brand фон (единствената тъмнозелена секция на страницата — контрастът ѝ дава тежест), бяло заглавие, една CTA в бяло.

### 5.2 Каталог (/shops, /products)
- Филтърна лента: sticky под header-а, pill-стил активни филтри с ✕, изчистване с един клик.
- ShopCard: лого/инициал в brand-soft кръг, име (h3), град + категория като тихи метаданни с икони 14px, описание 2 реда clamp. Hover патърнът от §3.3.
- CatalogProductCard: снимка 4:5 (image zoom hover), промо цена: стара цена зачертана `ink-faint` + нова в `accent` + badge „-X%".
- Празни резултати: EmptyState с line-art илюстрация + „Изчисти филтрите" действие.
- Пагинация: числова, с `rel=prev/next` за SEO.

### 5.3 Storefront (трите теми)
- Всички секционни компоненти минават полиране: типографската скала от §2.3 (мапната към `--sf-*` променливи), section paddings по темата, reveal анимации, consistent gap-ове.
- Hero слайдшоу: crossfade 600ms + Ken Burns `scale(1 → 1.06)` за времето на слайда (изключен при reduced-motion).
- Продуктова страница: галерия с thumbnail лента; variant смяна = crossfade на снимките + tick на цената; „Добави" с ✓ състоянието; свързани продукти със stagger.
- Trust лента под fold: методи на плащане/доставка на магазина като тихи икони — доверието се показва, не се разказва.
- Количка/checkout: най-строгата типография на сайта; totals с tnum; едит на ред количество със spring на числото; съобщенията за edge cases (изчерпано, сменена цена) в `accent-soft`/`danger-soft` карти per ред — ясни, не паникьосващи.

### 5.4 Dashboard
- Плътност: по-компактна скала (body 0.875rem), таблици с редове 52px, sticky header на таблиците.
- Sidebar: активен елемент = brand-soft фон + brand текст + 3px лява brand лента (motion `layoutId` — лентата се плъзга между елементите при навигация; малък, скъп на вид детайл).
- Статистически карти: числото едро (Onest 700, tnum), sparkline където има данни, delta стрелка в success/danger.
- Всички списъци със skeleton loading; всички деструктивни действия през ConfirmDialog (Radix AlertDialog основа).
- Dark mode: описаната палитра; toggle-ът с плавен crossfade на цветовете (`transition: background-color, color, border-color 300ms` на root — но само при toggle, не на page load, за да няма FOUC мигане).

### 5.5 Onboarding (§13)
- Прогрес: 4 стъпки с чекове, активната със spring пулс еднократно.
- Всяка стъпка завършва с micro-празник: ✓ анимация (SVG path draw, 400ms). При „Публикувай" — еднократен фин confetti burst в brand/accent цветове (canvas-confetti, 1.5s, никога повече от веднъж). Това е емоционалният връх на продукта — търговецът трябва да го почувства.

## 6. Компонентна основа: Radix под нашите примитиви

Не мигрираме към готова библиотека (би се била с tokens). Вместо това интерактивните примитиви получават **Radix Primitives** ядро (headless, accessibility наготово), визуалният слой остава 100% наш върху токените:

| Компонент | Radix основа | Какво печелим |
|---|---|---|
| Modal / ConfirmDialog | Dialog / AlertDialog | Focus trap, scroll lock, Escape, ARIA |
| SlidePanel | Dialog + custom анимация | Същото + bottom-sheet поведение |
| Select | Select | Keyboard навигация, typeahead |
| Tabs | Tabs | Roving focus, ARIA |
| Accordion | Accordion | ARIA + анимируема височина |
| Dropdown меню (нов) | DropdownMenu | Нужен за админ ред-действия |
| Tooltip (нов) | Tooltip | Dashboard икони |
| Switch (нов) | Switch | Настройки, „само промоции" toggle |

Останалите (Button, Input, Card, Badge, Table…) остават чисто наши. Всеки компонент: focus-visible ring (2px brand, offset 2px), 44px touch targets, loading/disabled/error състояния — както е по §7 от MVP спеца, но сега проверено ревизионно за всеки.

## 7. Визуални активи и изображения

- **Line-art икони система:** lucide-react (вече наличен) с stroke 1.75, размер 20/24, винаги `ink-soft` или `brand` — никога многоцветни икони.
- **EmptyState илюстрации:** 5-6 ръчно направени SVG line-art сцени (празна количка, няма продукти, няма резултати, няма поръчки, 404) — stroke в `line-strong`, един акцентен детайл в `brand`. Малки (<3 KB), в бранд, никога не изглеждат като стоков клипарт.
- **Демо магазини:** курирани фото серии с еднаква цветова температура per ниша (топли earthy тонове за „Ферма Зелена долина", неутрални студийни за „Глоу Козметика", текстурни близки планове за „Ателие Ръчичка"). Всяка снимка минава през еднакъв crop (4:5 продукти, 21:9 hero). Това е най-евтиният начин каталогът да изглежда като жив пазар, а не като тестова среда.
- **OG images:** генерирани 1200×630 per тип страница (landing, магазин, продукт, статия) — brand фон верзия на палитрата, Onest заглавие, лого. Динамичните (магазин/продукт) през `@vercel/og` (ImageResponse) — вече сме на Vercel, нула допълнителна инфраструктура.
- **`next/image`** навсякъде с `sizes`, blur placeholder за hero/големи изображения.

## 9. Performance бюджет (визуалният редизайн не може да оскъпи зареждането)

| Метрика | Бюджет |
|---|---|
| LCP (landing, mobile) | < 2.0s |
| CLS | < 0.02 (шрифтове self-hosted + size-adjust; hero витрината с фиксирани размери) |
| JS добавен от редизайна | Motion ~34 KB gz (tree-shaken `m` компонент + LazyMotion `domAnimation`); Radix ~15 KB; confetti lazy-loaded само в onboarding |
| Снимки | AVIF/WebP през next/image; hero витрината е DOM, не media |
| Lighthouse (mobile, landing) | ≥ 95 performance / 100 accessibility |

Motion се зарежда през `LazyMotion` + `m.` компоненти (не пълния `motion.`) — това сваля бандъла почти наполовина и е задължителна конвенция в кодбазата (ESLint правило срещу директен `motion.` импорт).

## 10. Имплементационни бележки

Редът на работа е дефиниран във **фазирането в §20** (R1 → R6 + C1) — то е задължителното разбиване; спецът не се изпълнява като един план. Общи бележки, валидни за всички фази:

- Полирането per повърхност върви в реда: landing → каталог → storefront секции/теми → checkout → dashboard (първо това, което продава).
- Hero витрината се разработва зад флаг и се мери отделно за performance преди включване.
- Гейтът на всяка фаза: `pnpm check` + релевантните точки от нейния DoD; пълният §19 се проверява след R5.
- Playwright скрийншот матрица: 360/768/1440px × 3 теми × light/dark — ръчен преглед при всяка фаза, докосваща визуалния слой.

**Задължителни ADR записи в `docs/decisions/`:** (1) смяна на Space Grotesk — липса на кирилица (отклонение от одобрения спец на План 3 → първо се обновява и той); (2) избор Motion пред GSAP; (3) Radix-под-собствени-примитиви подходът; (4) всеки одобрен R6 ефект — с резултата от прегледа „с/без".

## 12. Responsive стратегия (fully responsive, без изключения)

- **Mobile-first, 4 breakpoint-а:** `base` (360–639px, дизайнира се първо), `sm` 640, `lg` 1024, `xl` 1280. Съдържателен контейнер: `max-width: 1200px` (landing hero: 1280px), странични отстояния `clamp(1rem, 4vw, 2.5rem)`.
- **Fluid, не стъпаловидно:** типографията (§2.3) и секционните paddings са с `clamp()` — layout-ът „диша" между breakpoint-ите вместо да скача. Grid-овете с `auto-fill/minmax` (продуктови карти `minmax(240px, 1fr)`), не с фиксиран брой колони.
- **Per-повърхност поведение:** dashboard sidebar → bottom navigation + SlidePanel под `lg`; таблици → карти-редове под `sm` (никога хоризонтален скрол на данни); филтри в каталога → bottom-sheet под `lg`; hero витрината → опростена версия (§4); галерията на продукта → swipe с scroll-snap.
- **Тъч правила:** 44px минимум за всичко интерактивно; hover ефектите имат тъч еквивалент (активното състояние) или са чисто декоративни; никаква функционалност само зад hover.
- **Проверка:** Playwright скрийншот матрицата (§10) включва 360px, 768px и 1440px — трите размера се преглеждат ръчно при всяка визуална промяна.

## 13. Пространство и композиция (елегантна употреба на whitespace)

Пространството е дизайн материал номер едно на „тихия премиум" — то, а не декорацията, създава усещането за скъпо.

- **8px базова мрежа:** всички отстояния са кратни на 4/8 (`--space-1: 4px` … `--space-24: 96px`). Нула произволни стойности — ритъмът се усеща дори когато не се вижда.
- **Йерархия чрез близост:** свързаните елементи са близо (заглавие↔подзаглавие: space-3), несвързаните — далеч (секция↔секция: clamp 4–7rem). Правилото: разстоянието МЕЖДУ групите е минимум 2× разстоянието ВЪТРЕ в групата.
- **Асиметрия с цел:** landing секциите не са всички центрирани — „Функции" блоковете са 5/7 колонна двойка с редуващи се страни; hero-то е 6/6 текст/витрина на desktop. Центрирано е само това, което е кратко (CTA секции, eyebrow заглавия).
- **Плътност per контекст:** маркетингът е просторен (въздухът продава), dashboard-ът е компактен (ефективността обслужва), storefront-ът е по средата и се влияе от темата (warm е най-просторна).
- **Негативното пространство около сигнатурния елемент:** hero витрината има минимум space-16 около себе си на desktop — спектакълът се рамкира с тишина.

## 14. Разширен ефектен слой (parallax, scroll ефекти, анимирани илюстрации, видео)

Надгражда речника от §3.3. Принципът остава: всеки ефект обслужва съдържанието; но слоят става по-богат — целта е „впечатляващо", постигнато с техника, не с шум. Всичко по-долу спазва reduced-motion (§3.1) и performance бюджета (§9).

| Ефект | Имплементация | Къде | Защо е добре, а не кич |
|---|---|---|---|
| **Мек parallax** | Motion `useScroll` + `useTransform`: фонови декоративни форми се движат с 0.85–0.92× скоростта на скрола (никога съдържание, само фон) | Landing hero фон, финалната CTA секция | Дълбочина без дезориентация; съдържанието остава стабилно |
| **Scroll-linked прогрес** | Тънка 2px brand линия под платформения header, `scaleX` по scroll прогреса | Блог статии | Ориентация в дълъг текст |
| **Заглавия с word-reveal** | H2 на landing секциите: думите влизат stagger-нато (opacity+y, 40ms/дума) при whileInView | Само landing (не storefront — там търговецът е героят) | Едрата типография получава момент, без SplitText зависимост |
| **Animated icons** | Lucide иконите в „Болката"/„Функции": SVG path-draw при влизане във viewport (`pathLength` 0→1, 600ms) + микро-движение на hover (напр. количката се „поклаща" 2°) | Feature карти, EmptyState | Иконата разказва, не стои |
| **Живи илюстрации** | EmptyState SVG-тата (§7) получават по 1 анимиран детайл на цикъл 4–6s (примигваща звезда, плаваща кутия ±4px) — CSS keyframes, пауза при reduced-motion | Празни състояния | Празният екран става покана, не задънена улица |
| **Ambient видео** | Едно (максимум!) фоново видео в секцията „Витрина" на landing: 6–8s loop, без звук, AV1/H.265 ≤1.5MB, `poster` кадър, lazy, спряно при reduced-motion и data-saver | Landing витрина секция | Живот в единствената секция, показваща реални магазини; бюджетът пази LCP |
| **Скрол-сглобяем „Как работи"** | Трите стъпки: при скрол всяка карта се „подрежда" с леко scale 0.96→1 + y, свързани с тънка SVG линия, която се дочертава между тях | Landing „Как работи" | Последователността се ПОЧУВСТВА като последователност |
| **Marquee доверие** | Безкраен бавен marquee (CSS, 40s цикъл, пауза на hover) с категориите ниши („Храни · Мода · Ръчна изработка · Козметика…") | Под hero | Запълва „социалното доказателство" преди да имаме лога на клиенти — честно и красиво |
| **Hover spotlight** | Radial-gradient overlay, следващ курсора върху pricing картите (CSS custom property + 1 mousemove listener) | Цени | Фин „погледни ме" на най-важната секция за конверсия |

**Стоп-правила** (за да е резултатът добър, както искаш): максимум 2 scroll-linked ефекта на страница; parallax никога върху текст; нищо не се движи постоянно във viewport-а освен marquee и ambient видеото; всяка нова идея за ефект се проверява срещу тезата от §0 — ако е „защото изглежда яко", а не „защото обяснява", отпада.

**Разделение по риск (виж фазирането в §20):** повечето ефекти в таблицата са базова хигиена на 2026 (path-draw икони, живи EmptyState детайли, marquee, scroll прогрес) и влизат с Motion ядрото. Четири обаче са в зоната, където „тих премиум" може да се обърне на „танцуващ сайт": **мек parallax, word-reveal заглавия, скрол-сглобяемият „Как работи" и hover spotlight** — те (плюс ambient видеото) са **опционални (фаза R6)** и се внедряват само поотделно, след визуален преглед на страницата БЕЗ тях: ако страницата вече впечатлява, ефектът не се добавя.

## 15. Цветна дълбочина: градиенти, overlays, атмосфера (без „сухи" плоски цветове)

Плоският цвят на големи площи изглежда евтин. Дълбочината се постига с фини преливания и слоеве — винаги в рамките на палитрата от §1, никога дъга.

- **Атмосферни фонови градиенти:** ключовите секции получават едва доловим radial glow вместо плътен фон:
  - Hero: `radial-gradient(ellipse 80% 60% at 70% 20%, rgb(20 102 90 / 0.07), transparent)` върху `--color-bg` — светлината „идва" откъм витрината.
  - Финална CTA (тъмната секция): `radial-gradient(ellipse at 50% 0%, #1B7A6C 0%, #14665A 45%, #0E4A40 100%)` + noise overlay — дълбоко, не плоско зелено.
  - Токенизирани: `--gradient-hero-glow`, `--gradient-cta`, `--gradient-card-sheen`.
- **Noise texture:** 2–3% opacity SVG noise (`feTurbulence`, инлайн data-URI, ~1KB) върху тъмните и градиентни повърхности — убива банерната „пластмасовост", придава материалност. Токен `--texture-noise`.
- **Image overlays:** всяка снимка с текст върху нея получава градиентен scrim (`linear-gradient(to top, rgb(28 36 32 / 0.65), transparent 55%)`) — четимост винаги, независимо от снимката на търговеца. Задължително в storefront hero секциите (търговците качват произволни снимки — scrim-ът е защитата на дизайна).
- **Border gradients:** Pro pricing картата и hero витрината: 1px градиентна рамка (`linear-gradient(135deg, #14665A, #C97B2D 120%)` през border-image/маска) — премиум сигнал, използван точно на 2 места.
- **Duotone бранд третиране:** маркетинговите снимки (не продуктовите!) могат да минат през фин duotone (`ink` сенки / `bg` светлини + 8% brand тинт) — кохезия на визуалния език.
- **Градиентен текст — забранен** освен точно едно място: числото „0 €" / ключова дума в hero, ако визуалният преглед покаже нужда от акцент. По подразбиране — не.
- **Сенки с цвят:** сенките под brand елементи са тонирани (`rgb(20 102 90 / 0.15)` вместо неутрално сиво) — топлината се пренася и в дълбочината.

## 16. Обяснителен слой (нищо неясно без обяснение)

Всичко потенциално неясно получава обяснение в правилния формат — това е част от дизайна, не от документацията:

- **Tooltip система (Radix Tooltip, §6):** всяка икона без етикет, всяко съкращение и всяка метрика в dashboard-а („Какво е конверсия?") има tooltip. Правило: няма икона-бутон без `aria-label` + tooltip.
- **Инлайн помощ във формите:** всяко нетривиално поле има помощен ред под етикета (напр. slug: „Това е адресът на магазина ви: frizmo.shop/s/вашето-име"). Сложните концепции (варианти, промоции, прагове за доставка) — с 1-изреченско обяснение + „Научи повече" линк.
- **Empty states = обучение:** празното състояние обяснява какво е това нещо, защо е полезно и какъв е първият ход („Още нямате промоции. Промоцията „купи 3 за 10 €" вдига средната поръчка. → Създай промоция").
- **Първи-път подсказки:** при първо влизане в таб (Уебсайт, Промоции) — dismissible карта с 2 изречения и мини-визуал какво може табът; състоянието се пази per потребител.
- **Микрокопи стандарт:** активен залог, изречението казва какво ще се случи („Публикувай магазина" → toast „Магазинът е публикуван"), грешките казват какво стана И как се оправя. Никакъв системен жаргон към търговеца (не „ISR ревалидация", а „Промените ще се видят до минута").
- **Публична страница „Как работи Frizmo Shops"** е извън MVP, но FAQ-то на landing-а поема тази роля — всеки въпрос-отговор е написан за човек без технически познания.
- **В кодбазата:** всеки нетривиален дизайн механизъм (scrim, noise, LazyMotion, contrast тест) има кратък коментар защо съществува — следващият разработчик (или AI агент) не гадае.

## 17. Модерни шрифтове — финален избор и защита

Изборът от §2 остава (Onest + Inter + Lora/Manrope) — той Е модерният избор за 2026 с твърдото ограничение кирилица. За протокола, отхвърлени алтернативи и защо:

| Кандидат | Защо не |
|---|---|
| Space Grotesk | Няма кирилица (сегашен бъг) |
| Clash Display / General Sans (Fontshare) | Няма кирилица |
| Geist | Кирилицата е непълна/нова; Vercel-асоцииран = „шаблонно" усещане |
| Golos Text | Силна кирилица, но по-безличен от Onest в display размери — остава резерва |
| Unbounded | Ефектен, има кирилица, но твърде декоративен за UI; допустим САМО ако при визуалния преглед hero-то иска повече характер — максимум за 1 дума |

Задължителни практики: variable font файлове (по 1 файл на семейство), `subsets: ['cyrillic','latin']`, `size-adjust` на fallback-а (нула CLS), `font-feature-settings: "tnum"` за всички числови контексти, `"ss01"/"cv"` алтернативи на Onest се преглеждат при имплементация за по-характерни а/у форми.

## 18. AI media и инструменти за създаване на активи

AI се използва като производствен инструмент под строг арт-дирекшън — никога „сурово" (суровите AI визуали се разпознават мигновено и убиват премиум усещането).

- **Демо магазини — снимков сет:** генериран AI фотореализъм (Midjourney/Flux клас инструменти) с **фиксиран style prompt per ниша** (обектив, светлина, цветова температура, фон), така че серията е консистентна като от една фотосесия: earthy топли за храните, студийни неутрални за козметиката, текстурни близки планове за ръчната изработка. Всяка снимка минава ръчен подбор + еднакъв crop/грейд (§7). Предимство пред Unsplash: нула повторения с чужди сайтове, точните продукти, които seed-ът описва, безсрочни права.
- **Line-art илюстрации (EmptyState, „Болката"):** AI генерира вариации по единен стилов prompt (stroke тегло, ъгъл, палитра от токените) → векторизация → ръчно изчистване до нашите SVG стандарти + анимираните детайли от §14.
- **OG images:** остават програмни (`@vercel/og`, §7) — детерминистични, винаги в бранд. AI не е нужен там.
- **Ambient видеото (§14):** AI видео генерация (Runway/Veo клас) по същия арт-дирекшън prompt ИЛИ стоков клип с бранд duotone грейд — избира се по-доброто при преглед; бюджетът ≤1.5MB е неотменим.
- **Иконни микроанимации:** ръчно (path-draw е тривиален с Motion). Ако потрябват по-сложни (напр. анимирано лого) — Rive или Lottie (LottieFiles AI генерация като чернова → ръчна редакция), lazy-loaded, само ако конкретен случай го оправдае.
- **Правила:** (1) всеки AI актив минава човешки преглед на бранд съответствие и артефакти (ръце, текстове в изображения — честите издайници); (2) никакви AI изображения на разпознаваеми хора; (3) prompt-овете се пазят в `docs/design/asset-prompts.md` — активите са възпроизводими и разширяеми в същия стил; (4) финалните файлове се оптимизират (AVIF/WebP, SVGO) като всички останали.

## 20. Фазиране на изпълнението (задължително — спецът НЕ се изпълнява като един план)

Спецът описва крайното състояние; работата се разбива на фази, всяка = самостоятелен имплементационен план в стила на roadmap-а (завършва с работещ, тестван, комитнат софтуер без TODO-та) и с индивидуално одобрение преди старт. Ред: строго R1 → R2 → R3 → R4 → R5; R6 е опционална; C1 тече паралелно и не блокира нищо.

| Фаза | Съдържание (секции) | Защо тук | Риск | DoD извадка |
|---|---|---|---|---|
| **R1: Основа** | Токени — палитри light/dark, радиуси, сенки, spacing мрежа (§1, §13-мрежата); шрифтове Onest/Inter/Lora/Manrope с кирилица (§2, §17); contrast unit тест; **ADR за смяната на Space Grotesk** | Най-голям визуален ефект за най-малко код — само стойности, нула нови зависимости | Нисък | Нула хардкоднати стойности; CLS < 0.02; contrast тест зелен |
| **R2: Дълбочина и пространство** | Градиенти/noise/scrim/тонирани сенки (§15); секционен ритъм и spacing одит (§13); полиране на карти, бутони, форми | Чисто CSS върху R1 токените | Нисък | Никакви плоски hero/CTA фонове; 8px мрежа навсякъде |
| **R3: Motion ядро** | Motion + LazyMotion + ESLint правилото; `lib/motion.ts`, `<Reveal>`/`<RevealList>`; базовият речник от §3.3 + нискорисковите ефекти от §14 (path-draw икони, живи EmptyState детайли, marquee, scroll прогрес, skeleton) | Първата нова зависимост — собствен план + performance проверка | Среден | Бандъл в бюджета (§9); reduced-motion пълно покритие |
| **R4: Radix основа** | §6 компонент по компонент, без промяна на външните API | Независим рефакторинг; axe sweep влиза тук | Среден | Всички примитиви с Radix ядро; axe зелен |
| **R5: Hero витрината** | Сигнатурният елемент (§4) + мини-UI визуалите на „Как работи"/„Функции" | Зависи от R1+R3; най-сложният компонент — сам в план | Висок | Пълна последователност desktop / съкратена mobile / статична reduced-motion; LCP в бюджет |
| **R6: Опционален ефектен слой** | 4-те рискови ефекта (§14) + ambient видео — **всеки поотделно, с индивидуално одобрение** след преглед на страницата без него | Глазура; може изобщо да не потрябва | Висок | Стоп-правилата спазени; преглед „с/без" документиран |
| **C1: AI активи** (паралелна, некодова) | §18: фото серии за демо магазините, line-art илюстрации, prompt документация | Content pass, не кодова задача; Unsplash снимките работят дотогава | — | Консистентни серии, ръчен преглед, prompt-ове в `docs/design/asset-prompts.md` |

**Правила:**
- R1+R2 сами решават „ужасния" сегашен вид (~70% от възприеманата промяна); R5 е това, което впечатлява; R6 се доказва, не се предполага.
- Всяко отклонение от този спец по време на имплементация → първо обновяване на спеца + ADR в `docs/decisions/`, после код (правилото от roadmap-а важи и тук).
- Дефиницията на готово в §19 е за ЦЕЛИЯ редизайн (след R5); всяка фаза гейтва само собствените си точки.
